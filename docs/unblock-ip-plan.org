#+TITLE: Unblock Individual IPs via Admin API
#+DATE: 2026-06-15

* Overview

Add an HTTP endpoint and admin CLI command to remove individual IPs from
Mizu's reputation tracker, allowing surgical unblocking without flushing
all caches. Expose the action in Fernrohr's blacklist UI as an "unblock"
button per row.

* Scope of "unblocking"

Removing an IP from the stats manager's =ips= map clears its reputation
score and =IsDenied= flag. This is sufficient to unblock the IP because:

- *Reputation checks* (=CheckIPReputation=) return neutral (0) for
  unknown IPs, so the IP is allowed through.
- *Rate limiting* (=RateLimiter=) uses sliding windows keyed on IP and
  is independent of reputation. Rate limit state is NOT cleared by this
  operation. If an IP is also rate-limited, that will expire naturally
  via the sliding window.
- *Connection tracking* (=ConnectionTracker= / =DistributedTracker=)
  counts active TCP connections and is independent of reputation. Not
  affected.

The admin CLI help text and API docs should clarify that =unblock-ip=
removes reputation state only. Rate limit windows and connection counts
are unaffected and expire independently.

* Buffered event re-creation

The stats manager processes events asynchronously via a buffered channel.
After =RemoveIP=, events already queued for that IP will call
=getOrCreateIP= and recreate the entry with a fresh slate. This means:

- The IP may briefly reappear in the tracker moments after unblocking.
- It starts with neutral reputation (0), so it will NOT be re-blocked
  unless it immediately generates new negative signals.
- This is acceptable behavior. The plan documents it here and in the API
  response message so operators are not surprised.

* Prerequisite: Normalize IPv6 on ingress

IPv6 addresses have multiple string representations. =net.SplitHostPort=
returns the raw string from the TCP connection, so the map key may be
non-canonical (e.g. =2001:0db8:85a3:0000:0000:8a2e:0370:7334= instead
of =2001:db8:85a3::8a2e:370:7334=).

Normalize once at the point IPs enter the system so all downstream
lookups, stats, and the new unblock endpoint work with consistent keys.

** Shared helper

Extract a =NormalizeIP= function used by both ingress and =RemoveIP=.
Returns =(string, bool)= so callers can distinguish invalid input from
a valid IP — this avoids the semantic split where =NormalizeIP= would
silently pass through garbage while =RemoveIP= rejects it.

#+begin_src go
// In pkg/stats/normalize.go (or similar shared location)

// NormalizeIP returns the canonical string form of an IP address and true,
// or ("", false) if the input is not a valid IP.
// For IPv6 this compresses zeros and lowercases hex digits.
func NormalizeIP(ip string) (string, bool) {
    parsed := net.ParseIP(ip)
    if parsed == nil {
        return "", false
    }
    return parsed.String(), true
}
#+end_src

* Stats manager: add =RemoveIP= method

** File: =pkg/stats/manager.go=

Add a public method that deletes a single IP entry from the reputation
map. Uses =NormalizeIP= so the caller doesn't need to worry about IPv6
format.

#+begin_src go
// RemoveIP removes an IP entry from the reputation tracker, effectively unblocking it.
// Returns true if the IP was found and removed, false if it was not tracked.
// Note: buffered events already in the channel may recreate the entry with a clean
// slate shortly after removal. This is harmless as the new entry starts with neutral
// reputation.
func (m *Manager) RemoveIP(ip string) bool {
    key, ok := NormalizeIP(ip)
    if !ok {
        return false
    }

    m.ipMu.Lock()
    defer m.ipMu.Unlock()
    _, exists := m.ips[key]
    if exists {
        delete(m.ips, key)
    }
    return exists
}
#+end_src

Thread safety: the event loop uses =getOrCreateIP= which also acquires
=ipMu=, so there is no data race. See "Buffered event re-creation"
above for the behavioral note on in-flight events.

*** Handler

- =POST= only
- Accept IP from query param (precedence) or JSON body (={"ip": "..."}=)
- Validate with =net.ParseIP= to reject garbage
- Return JSON with =removed= boolean for script-friendly consumption
- Log the action at INFO level (security-relevant admin operation)
- No additional rate limiting beyond basic auth (acceptable for v1)
- Requires =net= import (=fmt=, =io=, =json= already imported)

#+begin_src go
func (s *Server) unblockIPHandler(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
        return
    }
    if s.ipUnblocker == nil {
        w.Header().Set("Content-Type", "application/json")
        w.WriteHeader(http.StatusNotImplemented)
        json.NewEncoder(w).Encode(map[string]any{
            "status": "error",
            "error":  "IP unblocking not configured",
        })
        return
    }

    // Parse IP from query param or JSON body
    ip := r.URL.Query().Get("ip")
    if ip == "" {
        var body struct {
            IP string `json:"ip"`
        }
        if r.Body != nil {
            if err := json.NewDecoder(io.LimitReader(r.Body, 1024)).Decode(&body); err != nil {
                w.Header().Set("Content-Type", "application/json")
                w.WriteHeader(http.StatusBadRequest)
                json.NewEncoder(w).Encode(map[string]any{
                    "status": "error",
                    "error":  "malformed request body",
                })
                return
            }
            ip = body.IP
        }
    }
    if ip == "" || net.ParseIP(ip) == nil {
        w.Header().Set("Content-Type", "application/json")
        w.WriteHeader(http.StatusBadRequest)
        json.NewEncoder(w).Encode(map[string]any{
            "status": "error",
            "error":  "valid ip parameter is required",
        })
        return
    }

    removed := s.ipUnblocker.RemoveIP(ip)

    // Log the admin action
    s.logger.Info("IP unblock requested",
        "ip", ip,
        "removed", removed,
        "remote_addr", r.RemoteAddr,
    )

    w.Header().Set("Content-Type", "application/json")
    msg := fmt.Sprintf("IP %s removed from reputation tracker", ip)
    if !removed {
        msg = fmt.Sprintf("IP %s was not tracked", ip)
    }
    json.NewEncoder(w).Encode(map[string]any{
        "status":  "success",
        "removed": removed,
        "message": msg,
    })
}
#+end_src

Always returns =200= for a valid request. The =removed= boolean
distinguishes "found and removed" from "was not tracked". This keeps it
simple for clients: =httpPost= in mizu-admin rejects non-2xx, so a 404
for "not tracked" would cause a =fatal()= error — wrong behavior for a
valid operation. Scripts check the =removed= field instead.

Response semantics:
- =200= + =removed: true= — IP was found and removed
- =200= + =removed: false= — IP was not in the tracker
- =400= — invalid or missing IP / malformed body
- =405= — wrong HTTP method
- =501= — unblocker not configured

func cmdUnblockIP() {
    if flag.NArg() < 2 {
        fmt.Fprintf(os.Stderr, "Usage: mizu-admin unblock-ip <ip-address>\n")
        os.Exit(1)
    }
    ip := flag.Arg(1)
    if net.ParseIP(ip) == nil {
        fatal("Invalid IP address: %s", ip)
    }

    bodyJSON, _ := json.Marshal(map[string]string{"ip": ip})
    resp, err := httpPost("/api/unblock-ip", bytes.NewReader(bodyJSON))
    if err != nil {
        fatal("Failed to unblock IP: %v", err)
    }

    var result map[string]any
    if err := json.Unmarshal(resp, &result); err != nil {
        fatal("Failed to parse response: %v", err)
    }

    removed, _ := result["removed"].(bool)
    msg, _ := result["message"].(string)
    if removed {
        fmt.Printf("OK: %s\n", msg)
    } else {
        fmt.Printf("Not found: %s\n", msg)
    }
}

#+begin_src go
// UnblockIP removes an IP from the Mizu reputation tracker.
func (c *MizuClient) UnblockIP(ctx context.Context, ip string) (bool, error) {
    bodyJSON, _ := json.Marshal(map[string]string{"ip": ip})
    req, err := http.NewRequestWithContext(ctx, http.MethodPost,
        c.baseURL+"/api/unblock-ip", bytes.NewReader(bodyJSON))
    if err != nil {
        return false, err
    }
    req.Header.Set("Content-Type", "application/json")
    if c.username != "" {
        req.SetBasicAuth(c.username, c.password)
    }
    resp, err := c.http.Do(req)
    if err != nil {
        return false, err
    }
    defer resp.Body.Close()

    var result struct {
        Removed bool   `json:"removed"`
        Message string `json:"message"`
    }
    if err := json.NewDecoder(resp.Body).Decode(&result); err != nil {
        return false, fmt.Errorf("decode: %w", err)
    }
    return result.Removed, nil
}
#+end_src

** BlacklistHandler: add POST handler for unblock action

*** File: =fernrohr/internal/handler/blacklist.go=

Add an =UnblockHandler= that the blacklist page's unblock buttons POST
to. It tries all Mizu nodes in the cluster (since reputation is
per-node) and reports results.

#+begin_src go
func (h *BlacklistHandler) UnblockHandler() http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        if r.Method != http.MethodPost {
            http.Error(w, "Method not allowed", http.StatusMethodNotAllowed)
            return
        }

        ip := r.FormValue("ip")
        if net.ParseIP(ip) == nil {
            http.Error(w, "Invalid IP", http.StatusBadRequest)
            return
        }

        cl := ActiveCluster(h.clusters, r)
        ctx, cancel := context.WithTimeout(r.Context(), blacklistTimeout)
        defer cancel()

        // Unblock on ALL Mizu nodes (reputation is per-node)
        var removed int
        var errs []string
        for _, mc := range cl.Mizus {
            ok, err := mc.UnblockIP(ctx, ip)
            if err != nil {
                h.logger.Warn("unblock: mizu node failed", "ip", ip, "error", err)
                errs = append(errs, err.Error())
                continue
            }
            if ok {
                removed++
            }
        }

        h.logger.Info("IP unblock via UI",
            "ip", ip,
            "cluster", cl.Name,
            "nodes_removed", removed,
            "nodes_failed", len(errs),
        )

        // Build redirect with flash message via query param
        dest := "/blacklist?"
        if q := r.URL.Query().Get("cluster"); q != "" {
            dest += "cluster=" + url.QueryEscape(q) + "&"
        }
        if len(errs) > 0 {
            dest += "msg=" + url.QueryEscape(
                fmt.Sprintf("Unblocked %s on %d node(s), %d failed", ip, removed, len(errs)))
        } else if removed > 0 {
            dest += "msg=" + url.QueryEscape(
                fmt.Sprintf("Unblocked %s on %d node(s)", ip, removed))
        } else {
            dest += "msg=" + url.QueryEscape(
                fmt.Sprintf("%s was not tracked", ip))
        }
        http.Redirect(w, r, dest, http.StatusSeeOther)
    }
}

* Cluster considerations

Each Mizu node tracks IP reputation independently. The =mizu-admin
unblock-ip= command only affects the node receiving the API call.

For clustered deployments:
- *Fernrohr* handles this automatically: =UnblockHandler= iterates all
  Mizu nodes in the cluster and calls =/api/unblock-ip= on each.
- *CLI users* must call =unblock-ip= on each node, or accept that the
  IP ages out of peer data via the S3 sync/retention cycle.

A cluster-wide unblock broadcast via memberlist gossip could be added
later but is not needed for v1.

* Design notes

** Input precedence

The =/api/unblock-ip= handler accepts IP from both query parameter and
JSON body. Query parameter takes precedence — if =?ip=1.2.3.4= is set,
the body is not read. This matches the pattern in =renewCertHandler=.

** =NormalizeIP= returns =(string, bool)=

Unlike a pass-through helper that returns the input unchanged on
failure, =NormalizeIP= returns =("", false)= for invalid IPs. This
ensures callers cannot accidentally use un-normalized garbage as a map
key. Both ingress (=server.go=) and =RemoveIP= use the same function,
keeping normalization behavior consistent.

** Always 200 for valid unblock requests

Returning 404 for "IP not tracked" was considered but rejected:
- The =httpPost= helper in =mizu-admin= treats non-2xx as fatal errors
- "Not tracked" is a valid outcome, not an error condition
- Scripts check the =removed= boolean field instead of HTTP status

** Existing =httpPost= sets Content-Type

The =httpPost= helper in =cmd/mizu-admin/main.go= already sets
=Content-Type: application/json= (line 1248). No change needed.

** Fernrohr =auth.BasicAuth= accepts =http.Handler=

=auth.BasicAuth(next http.Handler, ...)= takes =http.Handler=.
=http.HandlerFunc= satisfies =http.Handler=, so passing the result of
=UnblockHandler()= (which returns =http.HandlerFunc=) works. This is the
same pattern used for =RDNSHandler()= at line 190 of =main.go=.

** =ActiveCluster= is a real helper

Defined in =handler/cluster.go:40=:
=func ActiveCluster(clusters []Cluster, r *http.Request) *Cluster=.
Already used by the existing =BlacklistHandler.ServeHTTP= (line 66).

** =clusterParam= is an existing template function

Registered in =cmd/fernrohr/main.go:80= as a template =FuncMap= entry.
Takes a string (active cluster name), returns =template.URL= like
=cluster=name&=. Used throughout all Fernrohr templates (nav, dashboard,
trace, log, queue, resolve). The blacklist template currently has no
links needing it, but the new unblock form action URL requires it.

** Wiring order: =SetIPUnblocker= before =Start()=

=setupRoutes= is called inside =Start()= and registers =/api/unblock-ip=
unconditionally. The handler checks =s.ipUnblocker == nil= at runtime.
=SetIPUnblocker= must be called before =Start()= — same as
=SetStatsProvider= (line 755) and =SetCacheFlusher=, both called inside
=startHealthServer= before =Start()= at line 762.

** IPs in =confirm()= dialog are safe

IP addresses contain only digits, hex chars, dots, and colons — no
single quotes or HTML-special characters. The =confirm('Unblock {{.IP}}?')=
template expression is safe without additional escaping.

** No rate limiting beyond basic auth (v1)

The unblock endpoint is protected by HTTP Basic Auth. No per-endpoint
rate limiting is added for v1. The blast radius is limited: worst case,
an attacker with valid admin credentials can flush reputation entries,
which is equivalent to the existing =flush-cache= command. If audit
requirements grow, add request logging to a structured audit log.
